# Root Directory CS Files Documentation

## Program.cs
**役割**: アプリケーションのエントリーポイント。単一インスタンスの強制とWindows Formsアプリケーションの初期化を担当。

### 関数詳細
- **Main()** (static void)
  - 処理: ミューテックスを使用して重複起動を防止。作成できなければメッセージボックスを表示して終了。アプリの初期化を行い、VSA_launcherフォームを実行。
  - 期待する値: なし (メインエントリーポイント)
  - 返す値: void
  - 追加処理: DEBUGモードではコンソールウィンドウを表示。

## Form1.cs
**役割**: メインのWindows Forms UIフォーム。スクリーンショット処理、OSC通信、VRChat連携などの全機能を統合。

### VSA_launcher クラス
#### コンストラクタ VSA_launcher()
- 処理: コンポーネント初期化、設定読み込み、OSC関連サービスの初期化、ファイル監視サービスの設定、イベントハンドラの登録、タイマーの開始。
- 期待する値: なし
- 返す値: なし
- 追加処理: 例外発生時はエラーメッセージを表示してアプリ終了。

#### LaunchMainApplication() (public void)
- 処理: メインアプリが開発中であることをメッセージボックスで通知。
- 期待する値: なし
- 返す値: void

#### UpdateLaunchButtonState() (private void)
- 処理: メインアプリの実行状態に基づいて起動ボタンの状態を更新。
- 期待する値: なし
- 返す値: void
- 追加処理: UIスレッドでの実行を保証。

#### IsMainAppRunning() (private bool)
- 処理: 指定されたプロセス名で実行中のプロセスを検索。
- 期待する値: なし
- 返す値: bool - プロセスが見つかればtrue

(他の関数も多数存在 - イベントハンドラ、タイマーコールバック、OSC関連処理、画像処理、設定管理など)
